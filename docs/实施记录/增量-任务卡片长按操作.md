# 任务卡片长按操作（计划外增量功能）

> 本文档记录「任务卡片长按 → 操作菜单」功能的实现与后续迭代修改。每次修改在「修改记录」追加一节。

## 功能目标

为首页任务列表项（TaskListItem）增加长按交互：长按任务卡片 500ms 弹出自定义操作菜单，提供「删除」「编辑」「取消」三个操作。

- **编辑**：跳转 TaskEditPage 编辑该任务（复用既有路由 `{ taskType, isAdd: false }`）。
- **删除**：经 AlertDialog 二次确认后逻辑删除任务（isOpen 置 0），保留历史打卡记录，任务可在添加页重新开启。
- **取消**：关闭弹窗。

该功能为 Phase 5 之后的计划外增量，不在主干 Phase 0–10 任务清单内，独立记录于此。依据 `docs/实施计划/待完成功能列表.md` 完成记录与 `docs/设计文档.md` 任务管理章节。

## 修改记录

### 迭代 1：长按功能初版（系统 ActionSheet）

**日期**：2026-07-09

**改动**：
- `TaskListItem` 加 `LongPressGesture(duration:500)` + `onItemLongPress` 回调成员。
- `HomeTab` `showTaskActionSheet` 调系统 `ActionSheet.show`（编辑/删除两项）+ `confirmDeleteTask` 调 `AlertDialog.show` 二次确认。
- `TaskRepository` 新增 `deleteTask`：`UPDATE task SET isOpen=0`（逻辑删除，保留 task_record 历史）。
- `HomeViewModel` 新增 `deleteTask`：调 Repository 后重载 taskList/todayRecords 并刷新 AppStorage。
- 新增 string 资源：`delete_task`、`delete_task_confirm`、`select_action`（base + en_US）。

**决策**：删除采用逻辑删除（isOpen=0）而非物理 DELETE。原因：task 表预置 6 行，物理删除会丢失历史打卡且影响 TaskAddPage 判断；逻辑删除后任务可重新添加，streak 存于 Prefs 不受影响。

**构建**：arkts_check 无 Error；build SUCCESS（3.3s）。

### 迭代 2：自定义美观弹窗（替换系统 ActionSheet）

**日期**：2026-07-09

**改动**：
- 移除系统 `ActionSheet.show`，改用 `bindContentCover` + `@Builder taskActionSheetBuilder` 自定义弹窗（与 ClockInDialog 同风格）。
- 布局：白卡 + 半透明遮罩 `#66000000`；顶部任务图标(40×40)+任务名居中；底部「删除/编辑/取消」三等宽胶囊按钮横排，间隔 12dp 居中。
- 点遮罩关闭：外层 Column onClick 置 false，白卡加 `hitTestBehavior(HitTestMode.Block)` 防止点卡片误关。
- 新增成员 `@State showTaskActionSheet: boolean`、`@State actionSheetTaskType: TaskType`；原方法 `showTaskActionSheet(task)` 改名 `openTaskActionSheet(task)` 避免与 state 重名。

**修复**：成员 state 与方法同名 `showTaskActionSheet` 导致 Duplicate identifier + 误把 boolean 当函数调用（arkts_check 报 3 个 Error）→ 方法改名 `openTaskActionSheet`。

**构建**：arkts_check 无 Error；build SUCCESS（2.3s）。

## 技术要点

- **长按手势**：`LongPressGesture({ repeat: false, duration: 500 }).onAction(() => {...})`，绑定在 TaskListItem 的根 Row 上，与 `.onClick`（触发打卡）并存，互不干扰。
- **自定义弹窗**：`bindContentCover($$show, builder)` 双弹窗叠加（打卡弹窗 + 操作菜单），各自独立 @State 控制。
- **遮罩点击关闭**：外层全屏 Column 设 onClick 关闭；内层白卡 `hitTestBehavior(HitTestMode.Block)` 阻断冒泡，避免点卡片内容误关。
- **逻辑删除语义**：`UPDATE task SET isOpen=0 WHERE taskType=?`。getOpenTasks 过滤 isOpen，TaskAddPage 按 isOpen 判断已添加，删除后任务从首页消失但可在添加页重新开启，历史 task_record 保留。

### 迭代 5：主题默认背景色改为 #F2F3F5

**日期**：2026-07-09

**目标色确认**：经 vision-helper 解析 `screenshots/healthyfile9.png`，页面主背景为浅冷灰 `#F2F3F5`（任务卡片/Tab 栏为纯白 `#FFFFFF`）。

**改动**：
- `entry/src/main/resources/base/element/color.json`：`start_window_background` 由 `#FFFFFF` 改为 `#F2F3F5`（启动窗口；dark 模式保持 `#000000`）。
- `view/home/HomeTab.ets`：根 Stack 显式加 `.backgroundColor('#F2F3F5')`。
- `view/mine/MineTab.ets`：根 Column 显式加 `.backgroundColor('#F2F3F5')`。
- `pages/TaskAddPage.ets`、`pages/TaskEditPage.ets`：页面背景 `#F5F5F5` → `#F2F3F5`。

**根因修正**：首版仅改 start_window_background 资源 + TaskAddPage/TaskEditPage，未给 HomeTab/MineTab 根容器设显式背景；而运行时页面背景不会自动套用 start_window_background（该资源仅影响启动窗口），导致主界面仍呈白色。本次补 HomeTab/MineTab 显式背景后生效。MainTab 根 Tabs 不加背景（Tab 栏保持纯白）；AchievementPage 黑底、Index 启动图不变。

**说明**：弹窗按钮背景 `#F5F5F5`（迭代 3 设定）未改动。

**构建**：build SUCCESS（2.8s）。

### 迭代 6：首页背景色修复（Tabs 默认白底根因）+ 周历卡片化 + 截图调试流程

**日期**：2026-07-09

**背景**：迭代 5 将 HomeTab 根 Stack 设为 `#F2F3F5`，但用户反馈主界面背景仍为白色。经模拟器截图 + vision-helper 像素级分析排查，定位根因。

**根因**：`MainTab.ets` 的 `Tabs` 组件有默认白色背景，覆盖了 HomeTab 根 Stack 的 `#F2F3F5`，导致整个内容区呈白底。迭代 5 仅改了 HomeTab/MineTab 根容器，未处理 Tabs 层。

**改动**：

| 文件 | 修改 | 说明 |
|---|---|---|
| `pages/MainTab.ets` | Tabs 加 `.backgroundColor('#F2F3F5')` | **根因修复**：Tabs 默认白底覆盖子页面灰底 |
| `view/home/HomeTab.ets` | 内层 Column 加 `.backgroundColor('#F2F3F5')` | 双保险，确保灰底穿透 |
| `view/home/HomeTab.ets` | List 加 `.backgroundColor('#00000000')` | 防 List 默认白底覆盖 |
| `view/home/HomeTab.ets` | 列表包裹 Column 加 `.backgroundColor('#F2F3F5')` | 同上 |
| `view/home/HomeTab.ets` | WeekCalendar 包裹白色卡片容器 | 白底 `#FFFFFF` + borderRadius 12 + shadow + margin 12，与设计稿一致 |
| `view/home/TaskListItem.ets` | 卡片加 `.shadow({ radius:4, color:'#1A000000', offsetY:1 })` | 增强卡片立体感，与设计稿白色卡片+阴影一致 |
| `entryability/EntryAbility.ets` | 新增 `setWindowBackground()` 方法 | `windowStage.getMainWindow()` → `setWindowBackgroundColor('#F2F3F5')`，窗口级灰底兜底 |
| `AGENTS.md` | 新增「截图调试流程」章节 | 标准化模拟器截图→vision-helper 分析→修正→关闭模拟器的调试闭环 |

**验证方法**：按 AGENTS.md 截图调试流程，在模拟器 Pura 上部署后截图（`hdc shell snapshot_display`，必须 `.jpeg`），用 PowerShell `System.Drawing` 读取实际像素值确认：

```
Emulator screenshot2.jpeg (1320×2856):
  pixel(50%, 60%) = R=242 G=243 B=245  hex=#F2F3F5  ✅ 灰底
  pixel(50%, 75%) = R=237 G=238 B=240  hex=#EDEEF0  ✅ 灰底
```

**构建**：build SUCCESS（2.5s）；arkts_check 无 Error（仅项目既有 deprecated + async-throw WARN）。

**遗留**：`#F2F3F5` 为极浅灰（RGB 242/243/245），与纯白差异约 5%，部分屏幕可能不易区分；如需更深灰底可后续调整为 `#EDEEF0` 或 `#E8E9ED`。
