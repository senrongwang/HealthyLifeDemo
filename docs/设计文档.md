## 1. 文档概述

### 1.1 应用定位

"健康生活"是一款帮助用户养成每日健康习惯的 HarmonyOS 应用。用户可创建最多 6 个健康任务（早起、喝水、吃苹果、每日微笑、刷牙、早睡）并设置目标，每日打卡完成，追踪连续天数并解锁成就。应用还提供桌面服务卡片（1x2 任务列表 / 2x2 进度）与早起/早睡通知提醒。

### 1.2 功能清单

| 编号 | 功能 | 说明 |
|---|---|---|
| F1 | 任务创建 | 最多 6 个任务，设置目标（喝水/吃苹果按量多次打卡；其余单次打卡即完成） |
| F2 | 每日打卡 | 主页对任务打卡，显示当天完成进度；全部完成进度 100% 且连续天数 +1 |
| F3 | 连续天数 | 当天全部打卡完成 → 连续打卡天数 +1 |
| F4 | 成就解锁 | 连续 3/7/30/50/73/99 天解锁对应成就，动画弹窗提示，可在成就页查看 |
| F5 | 历史查看 | 查看以前健康任务完成情况（周历视图） |
| F6 | 添加任务流程 | 打开应用 → 主页 → 点 + 添加 → 任务列表显示已添加任务 |
| F7 | 服务卡片 1x2 | 桌面卡片显示已添加任务 |
| F8 | 服务卡片 2x2 | 桌面卡片显示任务完成进度 |
| F9 | 卡片拉起 | 点击 1x2/2x2 卡片拉起主页，看到任务列表 |
| F10 | 卡片定时更新 | 卡片配置更新时间，到点后次日重置任务 |
| F11 | 通知提醒 | 仅限早起、早睡，可设置提醒时间 |

本应用 ArkTS 编码实现者、评审人员。

## 2. 整体架构（MVVM）

### 2.1 三层职责

采用 **MVVM（Model–View–ViewModel）** 分层：

```
┌─────────────────────────────────────────────────────────┐
│  View（视图层）   pages/ + view/                         │
│  纯渲染 + 用户事件回调；不直接访问 RDB/Prefs            │
│  通过 @State/@ObjectLink/@Consume 绑定 ViewModel 状态    │
├─────────────────────────────────────────────────────────┤
│  ViewModel（视图模型层） viewmodel/                     │
│  页面级状态与业务逻辑；持有 @State 字段并写入 AppStorage│
│  调用 Repository 完成数据读写；不持有 UI 引用           │
├─────────────────────────────────────────────────────────┤
│  Model（模型层） model/ + data/                         │
│  实体类（@Observed）+ 数据访问（RDB/Prefs/Repository）  │
└─────────────────────────────────────────────────────────┘
```

### 2.2 数据流

```
用户操作 → View 事件回调 → ViewModel 方法 → Repository（RDB/Prefs）
                                                      ↓ 回调数据
View 重新渲染 ← @State/@ObjectLink 变更 ← ViewModel 更新状态
                                                      ↓
                                          AppStorage 同步全局态
                                                      ↓
                                          （成就弹窗/卡片刷新）
```

**单向依赖**：View → ViewModel → Model/Repository；Model 层不反向依赖 View/ViewModel。全局共享状态经 `AppStorage` 中转，ViewModel 既是 AppStorage 的读写者，也是 @State 的持有者。

### 2.3 与传统 MVC 的区别

- 无 Activity/Fragment 式控制器；页面（`@Entry/@Component`）只负责渲染与事件分发。
- 业务逻辑集中在 ViewModel，可被主页与卡片场景复用（卡片场景通过 Repository 直读数据，不经 ViewModel，见 §10.4）。
- 状态变化由 ArkUI 声明式框架自动驱动 UI 刷新，无需手动 `notifyDataSetChanged`。

## 3. 目录结构

采用 **2 模块**架构：`common`（HAR 静态共享库，公共能力层）+ `entry`（HAP 入口模块，产品层）。依赖严格单向：`entry → common`。

> **选型依据**：`UIAbility`/`ExtensionAbility` 只能在 HAP（entry/feature）的 `module.json5` 中声明，HAR/HSP 不支持声明 ExtensionAbility。故 `EntryAbility`、`EntryBackupAbility`、`FormAbility` 必须留在 `entry`。`common` 作为 HAR 集中存放 Model/Data/Utils 与共享资源，被 `entry` 编译期引用。卡片经 `common` 的 `TaskRepository` 取数、不依赖 `entry` 的 ViewModel，模块边界强制落实 §6.4 的上下文隔离。

```
项目根/
├── common/                              # 公共能力层（HAR 静态共享库）
│   ├── Index.ets                        # 统一导出对外接口（model/data/common）
│   ├── oh-package.json5                 # name:"common"
│   └── src/main/
│       ├── ets/
│       │   ├── model/                   # TaskType/TaskInfo/TaskRecord/AchievementInfo/DayProgress
│       │   ├── data/
│       │   │   ├── db/                  # RdbHelper/RdbTables
│       │   │   ├── prefs/               # PrefsHelper
│       │   │   └── repository/          # TaskRepository（数据门面）
│       │   └── common/                  # Constants/Logger/DateUtils
│       └── resources/                   # 共享资源 media/string/color/float（已存在，保留）
│
└── entry/                               # 产品层（HAP entry 入口模块）
    ├── oh-package.json5                 # dependencies: { "common": "file:../common" }
    └── src/main/
        ├── module.json5                 # type:"entry"；abilities(EntryAbility)
        │                                #  +extensionAbilities(EntryBackup/FormAbility)+requestPermissions
        ├── ets/
        │   ├── entryability/EntryAbility.ets
        │   ├── entrybackupability/EntryBackupAbility.ets
        │   ├── pages/                   # Index(Splash)/MainTab/HistoryPage
        │   ├── view/                    # home/achievement/mine/component/dialog
        │   ├── viewmodel/               # Home/Achievement/Mine/Task/History ViewModel
        │   ├── card/                    # form/FormAbility + widget/WidgetCard1x2/2x2
        │   └── service/ReminderHelper.ets
        └── resources/base/
            ├── element/                 # entry 专属：EntryAbility_label/desc、module_desc 等
            └── profile/                 # main_pages.json + form_config.json
```

> **资源策略**：共享 UI 资源（任务/徽章/Tab 图标、通用 string/color/float）保留在 `common` HAR，由 `entry` 与卡片经 `$r` 引用；`entry` 自身仅保留 `module.json5` 配置引用所需的少量资源（EntryAbility 名称/描述、应用图标 layered_image/startIcon、启动窗底色 start_window_background）。
>
> `pages/Index.ets` 改造为启动页（Splash），倒计时/首启权限弹窗后跳转 `MainTab`；`main_pages.json` 相应追加页面路径（见 §14）。
>
> 当前工程为单一 `common` entry 模块，实现时需按上述结构拆分：新建 `entry` 模块并迁入 Ability/页面/卡片/服务，将 `common` 转为 HAR（仅留 model/data/common/utils 与共享资源）。

## 4. 数据模型（Model）

### 4.1 枚举与常量

```arkts
// model/TaskType.ets
export enum TaskType {
  GETUP = 0,   // 早起（单次，可提醒）
  DRINK = 1,   // 喝水（多次，单位：次）
  APPLE = 2,   // 吃苹果（多次，单位：个）
  SMILE = 3,   // 每日微笑（单次）
  BRUSH = 4,   // 每日刷牙（单次）
  SLEEP = 5    // 早睡（单次，可提醒）
}

// 打卡标记：单次完成 vs 多次累计
export enum TaskFlag {
  SINGLE = 0,  // 打卡一次即完成
  MULTI = 1    // 按 targetValue 多次打卡
}

// 成就解锁阈值（天）
export const ACHIEVEMENT_THRESHOLDS: number[] = [3, 7, 30, 50, 73, 99];

// 是否支持提醒（仅早起/早睡）
export function isReminderSupported(type: TaskType): boolean {
  return type === TaskType.GETUP || type === TaskType.SLEEP;
}
```

### 4.2 实体类

```arkts
// model/TaskInfo.ets
import { TaskType, TaskFlag } from './TaskType';

@Observed
export class TaskInfo {
  taskType: TaskType;
  isOpen: boolean;          // 是否已添加
  targetValue: number;      // 目标量（单次任务为 1）
  flag: TaskFlag;           // 单次/多次
  reminderEnable: boolean;  // 提醒开关（仅 getup/sleep 有意义）
  reminderTime: string;     // "HH:MM"（仅 getup/sleep）

  constructor(taskType: TaskType, flag: TaskFlag) {
    this.taskType = taskType;
    this.flag = flag;
    this.isOpen = false;
    this.targetValue = flag === TaskFlag.SINGLE ? 1 : 1;
    this.reminderEnable = false;
    this.reminderTime = '08:00';
  }
}
```

```arkts
// model/TaskRecord.ets
import { TaskType } from './TaskType';

@Observed
export class TaskRecord {
  date: string;        // "YYYY-MM-DD"
  taskType: TaskType;
  finNum: number;      // 已完成次数
  targetValue: number; // 当天目标量（冗余存储，便于历史还原）

  constructor(date: string, taskType: TaskType, targetValue: number) {
    this.date = date;
    this.taskType = taskType;
    this.targetValue = targetValue;
    this.finNum = 0;
  }

  get isDone(): boolean {
    return this.finNum >= this.targetValue;
  }
}
```

```arkts
// model/AchievementInfo.ets
export class AchievementInfo {
  level: number;        // 顺序 0..5
  days: number;         // 阈值天（3/7/30/50/73/99）
  unlocked: boolean;
  iconOn: Resource;     // ic_badge_*_on
  iconOff: Resource;    // ic_badge_*_off

  constructor(level: number, days: number, iconOn: Resource, iconOff: Resource) {
    this.level = level;
    this.days = days;
    this.iconOn = iconOn;
    this.iconOff = iconOff;
    this.unlocked = false;
  }
}
```

```arkts
// model/DayProgress.ets
export class DayProgress {
  date: string;
  weekIndex: number;    // 0..6（周一..周日）
  doneCount: number;    // 当天已完成任务数
  totalCount: number;   // 当天应完成任务数
  isAllDone: boolean;

  constructor(date: string, weekIndex: number) {
    this.date = date;
    this.weekIndex = weekIndex;
    this.doneCount = 0;
    this.totalCount = 0;
    this.isAllDone = false;
  }

  get progress(): number {
    return this.totalCount === 0 ? 0 : Math.floor(this.doneCount * 100 / this.totalCount);
  }
}
```

## 5. 数据存储设计

### 5.1 关系型数据库（RDB）

使用 `@kit.ArkData`（`@ohos.data.relationalStore`）。数据库名 `HealthyLife.db`。

#### 表 1：`task`（任务定义，最多 6 行）

| 列 | 类型 | 约束 | 说明 |
|---|---|---|---|
| taskType | INTEGER | PRIMARY KEY | TaskType 枚举值 |
| isOpen | INTEGER | NOT NULL DEFAULT 0 | 0 未添加 / 1 已添加 |
| targetValue | INTEGER | NOT NULL DEFAULT 1 | 目标量 |
| flag | INTEGER | NOT NULL | 0 单次 / 1 多次 |
| reminderEnable | INTEGER | NOT NULL DEFAULT 0 | 提醒开关 |
| reminderTime | TEXT | | "HH:MM"（仅 getup/sleep） |

建表 SQL：

```sql
CREATE TABLE IF NOT EXISTS task (
  taskType       INTEGER PRIMARY KEY,
  isOpen         INTEGER NOT NULL DEFAULT 0,
  targetValue    INTEGER NOT NULL DEFAULT 1,
  flag           INTEGER NOT NULL,
  reminderEnable INTEGER NOT NULL DEFAULT 0,
  reminderTime   TEXT
);
```

#### 表 2：`task_record`（每日打卡记录）

| 列 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | |
| date | TEXT | NOT NULL | "YYYY-MM-DD" |
| taskType | INTEGER | NOT NULL | |
| finNum | INTEGER | NOT NULL DEFAULT 0 | 已完成次数 |
| targetValue | INTEGER | NOT NULL | 当天目标（冗余） |

建表 SQL + 唯一索引（同一日期同一任务一行）：

```sql
CREATE TABLE IF NOT EXISTS task_record (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  date        TEXT NOT NULL,
  taskType    INTEGER NOT NULL,
  finNum      INTEGER NOT NULL DEFAULT 0,
  targetValue INTEGER NOT NULL
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_date_task ON task_record(date, taskType);
```

#### RdbHelper 要点

- 单例（`static instance`），`getRdbStore(context)` 传入 `UIAbilityContext` / `formContext`。
- `onCreate` 中执行两条建表 SQL 并**预置 6 行 task 默认数据**（isOpen=0）。
- 异步 API 一律 `async/await`，回调结果显式类型，禁 `any`。

### 5.2 首选项（Preferences）

使用 `@kit.ArkData`（`@ohos.data.preferences`），store 名 `healthy_life_prefs`。

| Key | 类型 | 含义 |
|---|---|---|
| `streak_days` | number | 当前连续打卡天数 |
| `last_complete_date` | string | 上次全部完成日期（用于跨天判断是否中断） |
| `total_days` | number | 累计全勤天数 |
| `highest_streak` | number | 历史最高连续天数 |
| `achievements_unlocked` | string | 已解锁阈值逗号串，如 "3,7,30" |
| `first_launch` | boolean | 是否首次启动（触发权限弹窗） |
| `current_date` | string | 应用当前所在日期（跨天重置基准） |

> 成就解锁状态也可单列 RDB 表，但量小且为标量，存首选项更轻；`achievements_unlocked` 解析为 `number[]`。

### 5.3 TaskRepository（数据门面）

`TaskRepository` 为单例，组合 `RdbHelper` + `PrefsHelper`，对 ViewModel/FormAbility 暴露统一接口：

| 方法 | 说明 |
|---|---|
| `init(context)` | 初始化 RDB/Prefs（EntryAbility onCreate 调用） |
| `getTasks(): TaskInfo[]` | 读取全部任务定义 |
| `upsertTask(task: TaskInfo)` | 新增/更新任务（isOpen/targetValue/reminder） |
| `getTodayRecords(): TaskRecord[]` | 当天打卡记录（无则自动建行） |
| `clockIn(taskType: TaskType): TaskRecord` | 打卡：finNum+1（单次封顶 1） |
| `isAllDoneToday(): boolean` | 当天是否全部完成 |
| `getDayProgress(date): DayProgress` | 单日进度 |
| `getWeekProgress(): DayProgress[]` | 本周 7 天进度（历史视图） |
| `getStreak(): number` | 连续天数 |
| `onAllDone()` | 全部完成时：streak+1、检测成就、更新卡片 |
| `checkAchievements(): number` | 比对阈值，返回新解锁阈值（0 表示无） |
| `maybeRollOverDate()` | 跨天重置：currentDate 变更时新建当天记录 |

## 6. 状态管理方案

### 6.1 AppStorage（全局共享）

AppStorage 为应用级单例，存放跨页/跨组件共享态。在 `EntryAbility.onWindowStageCreate` 初始化前注入。

| Key | 值类型 | 含义 |
|---|---|---|
| `currentDate` | string | 当天日期 |
| `streakDays` | number | 连续天数 |
| `todayAllDone` | boolean | 当天是否全勤 |
| `taskList` | TaskInfo[] | 当前任务定义（@Observed 数组） |
| `todayRecords` | TaskRecord[] | 当天记录（@Observed 数组） |
| `achievementList` | AchievementInfo[] | 成就列表 |
| `splashDone` | boolean | 启动页是否结束 |

```arkts
// 注入示例（EntryAbility）
AppStorage.setOrCreate<string>('currentDate', DateUtils.today());
AppStorage.setOrCreate<number>('streakDays', 0);
AppStorage.setOrCreate<boolean>('todayAllDone', false);
```

### 6.2 @Observed + @ObjectLink（实体联动）

- `TaskInfo`、`TaskRecord` 标注 `@Observed`（见 §4.2）。
- `TaskListItem` 组件用 `@ObjectLink task: TaskInfo` / `@ObjectLink record: TaskRecord` 接收单条，属性变化自动刷新该项。
- 父组件（`HomeTab`）持有 `@State taskList: TaskInfo[]`，数组元素为 `@Observed` 对象；**注意**：ArkUI 中数组整体替换或元素属性变更才能触发刷新，新增/删除需整体赋值。

### 6.3 @Provide + @Consume（跨层级下发）

`HomeTab` 作为数据提供方：

```arkts
@Provide('homeVM') homeVM: HomeViewModel;
@Provide('todayProgress') todayProgress: DayProgress;
```

子组件 `ProgressRing`、`WeekCalendar`、`TaskListItem` 通过 `@Consume` 取用，避免逐层透传。

### 6.4 卡片场景特殊说明

ArkTS 服务卡片运行于**独立上下文（form 进程）**，**不能直接访问主页 AppStorage**。卡片数据由 `FormAbility` 调 `common` 模块的 `TaskRepository` 直读，再通过 `formProvider.updateForm` 传递 JSON 给卡片 UI（见 §10.4）。2 模块架构下，卡片代码仅 `import 'common'`、**不依赖 `entry` 的 ViewModel**，模块边界从工程层面强制此隔离。

## 7. 页面 View 设计

### 7.1 启动页 SplashPage（`pages/Index.ets`）

- 背景 `ic_splash_bg.png` 全屏；右上角 `skip_ads`（"跳过广告 %d s"）倒计时，到点或点击跳转 `MainTab`。
- 首启（`first_launch=true`）弹权限说明 `privacy_title` / `privacy_desc`，按钮 `confirm` / `cancel`；`confirm` 后置 `first_launch=false`。
- 倒计时结束 `AppStorage.set('splashDone', true)` 并路由到 `pages/MainTab`。

### 7.2 主页 MainTab（`pages/MainTab.ets`，Tabs 容器）

- `Tabs` + `TabContent` ×3，底部 `position` 固定。
- 三个 Tab 图标：`ic_tabs_home_normal/sel`、`ic_tabs_achievement_normal/sel`、`ic_tabs_mine_normal/sel`；文案 `tab_home` / `tab_achievement` / `tab_mine`。
- 默认选中 Home。

### 7.3 首页 HomeTab（`view/home/HomeTab.ets`）

布局自上而下：

1. **头部**：`ic_home_bg.png` 背景 + 标题 `healthy_life` + `ProgressRing`（显示 `target_progress` 百分比）。

### 9.1 ViewModel 骨架

```arkts
// entry/src/main/ets/viewmodel/HomeViewModel.ets
import { TaskInfo, TaskRecord, DayProgress, TaskType, TaskRepository } from 'common';

export class HomeViewModel {
  taskList: TaskInfo[] = [];
  todayRecords: TaskRecord[] = [];
  todayProgress: DayProgress = new DayProgress('', 0);

  async loadTodayTasks(): Promise<void> {
    this.taskList = await TaskRepository.getInstance().getTasks();
    this.todayRecords = await TaskRepository.getInstance().getTodayRecords();
    this.refreshProgress();
    AppStorage.setOrCreate<TaskInfo[]>('taskList', this.taskList);
  }

  async clockIn(taskType: TaskType): Promise<void> {
    const rec: TaskRecord =
      await TaskRepository.getInstance().clockIn(taskType);
    this.todayRecords = this.replaceRecord(rec);
    this.refreshProgress();
    if (this.todayProgress.isAllDone) {
      await this.onAllDone();
    }
  }

  refreshProgress(): void {
    const open = this.taskList.filter((t: TaskInfo): boolean => t.isOpen);
    const done = this.todayRecords.filter((r: TaskRecord): boolean => r.isDone);
    this.todayProgress.totalCount = open.length;
    this.todayProgress.doneCount = done.length;
    this.todayProgress.isAllDone =
      open.length > 0 && done.length === open.length;
  }

  private async onAllDone(): Promise<void> {
    await TaskRepository.getInstance().onAllDone();
    const newDays: number =
      await TaskRepository.getInstance().checkAchievements();
    if (newDays > 0) {
      this.showAchievementDialog(newDays);
    }
  }

  private showAchievementDialog(days: number): void {
    // 调用全局 openCustomDialog，渲染 AchievementDialog
  }

  private replaceRecord(rec: TaskRecord): TaskRecord[] {
    const arr: TaskRecord[] = this.todayRecords.slice();
    const idx: number = arr.findIndex(
      (r: TaskRecord): boolean => r.taskType === rec.taskType
    );
    if (idx >= 0) {
      arr[idx] = rec;
    } else {
      arr.push(rec);
    }
    return arr;
  }
}
```

import { relationalStore } from '@kit.ArkData';
import { common } from '@kit.AbilityKit';

export class RdbHelper {
  private static instance: RdbHelper | null = null;
  private store: relationalStore.RdbStore | null = null;
  private readonly DB_NAME: string = 'HealthyLife.db';

  static getInstance(): RdbHelper {
    if (RdbHelper.instance === null) {
      RdbHelper.instance = new RdbHelper();
    }
    return RdbHelper.instance;
  }

  async getStore(context: common.Context): Promise<relationalStore.RdbStore> {
    if (this.store !== null) {
      return this.store;
    }
    const config: relationalStore.StoreConfig = {
      name: this.DB_NAME, securityLevel: relationalStore.SecurityLevel.S1
    };
    this.store = await relationalStore.getRdbStore(context, config);
    return this.store;
  }

## 12. 关键业务流程

### 12.1 添加任务

```
点 ic_home_add → openCustomDialog(TaskDialog 添加模式)
  → 选任务 + 设目标(+时间/开关) → 点 complete
  → TaskViewModel.save → Repository.upsertTask
  → (getup/sleep) ReminderHelper 同步
  → AppStorage.taskList 刷新 → HomeTab 列表重渲染
```

### 12.2 打卡 + 进度更新

```
点 clock_in → HomeViewModel.clockIn
  → Repository.clockIn(finNum+1，单次封顶1)
  → 更新 todayRecords → refreshProgress(重算 done/total/progress)
  → 若 isAllDone → onAllDone:
       streak+1 → checkAchievements(比对阈值)
       → 命中新阈值 → openCustomDialog(AchievementDialog 动画)
       → 写 achievements_unlocked
       → formProvider.updateForm 刷新卡片
```

### 12.3 连续天数判定

```
当天全勤 → 读 last_complete_date:
  若为昨天 → streak+1（连续）
  若为今天（重复触发，幂等保护）→ 不变
  否则 → streak=1（中断重置）
更新 last_complete_date=今天、highest_streak、total_days
```

### 12.4 卡片刷新

```
onUpdateForm(系统周期/次日00:00触发)
  → Repository.maybeRollOverDate(跨天则新建当天记录)
  → 读取任务/进度 → updateForm(formId, json)
桌面卡片重渲染（2x2 显示次日 0% 进度，1x2 显示任务清单）
```

### 12.5 次日重置

跨天由 `current_date` 变化触发（应用前台 onShow 或卡片 onUpdateForm）：为当天每个 isOpen 任务在 `task_record` 建新行（finNum=0），历史行保留。任务定义（`task` 表）不变。
